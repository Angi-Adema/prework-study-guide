HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The importance of h1 through h6 headings and using them in order.
  • The img and src attibutes and the image within a page.
  • The significance of the footer element that appears at the bottom of the page.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • We can combine selectors by using a comma in order to minimize repeat coding lines.
  • The "*" applies rules to all elements visible on a page.
  • We can apply rules to only certain elements by using "." ahead of an element name and all elements with this name will show the applied rules.
  • Hexadecimal color codes can be looked up online in order to customize the colors and effects on the page.

Git

  • git status: Checks what branch we are currently on.
  • git checkout -b branch-name: Creates a new branch and switches to it.
  • cd "File Name": Allows you to switch to a different file in the directory.
  • git checkout -b "File Name": Creates a new branch to hold our work.
  • git add -A: Adds modifications to the current working branch and informs git the changes will be added in the next commit.

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An array contains several items using zero indexing where the first item holds position 0.
  • A function always starts with the keyword function and the code in the curley brackets is executed until the condition is met.
  • A for loop allows us to log several items at a time without having to type out the code for each item at a time.
